# Self-Learning Cycle: The Infinite Upward Spiral

このワークフローは、**「SECIモデル」**と**「ダブルループ学習」**に基づき、未知の事象やエラーを新たな知識（ルール・スキル・ワークフロー）へと変換するプロセスである。

## 1. Detection Phase (Trigger)

**状況:** 以下のいずれかが発生した場合

- 既存のルール/スキルで対応できないタスクに遭遇した。
- 実行結果が期待と乖離した（エラー、不整合）。
- ユーザーから明示的なフィードバックや修正指示を受けた。

**手順:**

1.  現在のタスクを「学習の種」として認識する。
2.  「なぜ既存のシステムでは対応できなかったのか？」を問いかける。

## 2. Exploration Phase (Hypothesis & Trial)

**状況:** Emergency Mode（補助輪）での対応
**手順:**

1.  **類似検索:** 過去のログや既存知識からヒントを探す。
2.  **仮説立案:** 論理的推論に基づき、暫定的な解決策（仮説）を立てる。
3.  **実験的実行:** 仮説を実行し、結果を詳細に観察する。
    - 成功した場合 → 次のフェーズへ
    - 失敗した場合 → 仮説を修正して再試行（または諦めて事実を記録）

## 3. Codification Phase (Externalization)

**状況:** 成功体験の言語化
**手順:**
以下のテンプレートを用いて、暗黙知を形式知に変換する。

```markdown
# Knowledge Seed

- **Context:** [どのような状況で？]
- **Problem:** [何が課題だったか？]
- **Solution:** [どう解決したか？]
- **Rationale:** [なぜうまくいったのか？（原理・法則）]
```

## 4. Systematization Phase (Combination & Internalization)

**状況:** システムへの統合
**手順:**
形式知化された知識を、適切なシステム要素に統合する。

1.  **ルールの更新 (`GEMINI.md`):**
    - 汎用的な行動指針や判断基準として追記・修正。
2.  **ワークフロー化 (`/creating-workflows`):**
    - 再現可能な手順として確立できる場合、新規WFを作成。
3.  **スキル化 (`/creating-skills`):**
    - 特定の技術やツールセットが必要な場合、新規Skillを作成。

## 5. Validation Phase

**手順:**

1.  ユーザーに統合結果（更新されたファイル等）を報告する。
2.  次回の類似タスクで、更新されたシステムが機能するか確認する。
